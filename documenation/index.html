Image Conversion API Documentation

Installation
Prerequisites
  • Node.js Version 16.x or above
  • MongoDB: A running instance for storing conversion job data and logs
Steps to install
1. Clone the repository:
git clone https://github.com/your-repo/image-conversion-api.git
cd image-conversion-api

2. Install dependencies:
npm install

3. Set up environment variables:
cp .env.example .env

4. Run the application:
npm run dev
API Key Management
Generate API Key

To generate a new API key, make a POST request to /api-keys/generate.

POST /api-keys/generate
Body:
{
  "name": "My Application Name"
}
Remove API Key

To remove an API key, make a DELETE request to /api-keys/{apiKey}.

DELETE /api-keys/{apiKey}
Single Image Conversion
Convert a Single Image

To convert a single image to WebP or AVIF, send a POST request to /convert.

POST /convert
Headers:
Authorization: Bearer 

Body (multipart/form-data):
image=
outputFormat=webp | avif
quality=80
compression=medium
Parameters
  • image: The image file to convert (multipart/form-data).
  • outputFormat: The desired output format (webp or avif).
  • quality: The quality of the converted image (integer from 1 to 100). Default is 80.
  • compression: The compression level (low, medium, high). Default is medium.
Response
{
  "status": "success",
  "message": "Image converted successfully.",
  "file": ""
}
Bulk Image Conversion
Bulk Image Conversion

For bulk conversion, send a POST request to /bulk-convert with multiple image files and parameters.

POST /bulk-convert
Headers:
Authorization: Bearer 

Body (multipart/form-data):
images=
outputFormat=webp | avif
quality=80
compression=medium
Response
{
  "jobId": "123456789",
  "message": "Bulk conversion job started successfully."
}
API Responses
Error Handling

When an error occurs, the API will return a message with a status code and description of the issue.

{
  "status": "error",
  "message": "Invalid image format"
}
Security

The API uses API keys for authentication. Make sure to store your API key securely and not expose it in public repositories or in client-side code.
